## 오늘 수정 내용

### 1. 🔴 모터 드라이버 EN 부팅 floating 차단 (물리 손상 방지)
- `setup()` 진입 **맨 첫 줄**에서 `PIN_EN=HIGH`(코일 차단). 기존엔 Serial+delay 뒤에야 설정 → 리셋~setup 구간(+크래시/WDT 리셋) 약 1초+ 동안 EN floating → 드라이버 enable·발열 위험.
- **양 버전 적용.**
- HW 병행 권장: **EN→3.3V 10K 풀업** (플래싱·무전원 구간까지 봉인).

### 2. 🟠 비상정지(STOP) 유실 방지
- `/api/stop` → 명령 큐(깊이8) 우회, `volatile g_estop` 플래그 직접 set. `loop()` 매 사이클 최우선 확인 → `emergencyStop()`. 큐 가득 차도 정지 안 잃음.
- **양 버전 적용.**

### 3. PT100 3선식 전환
- `MAX31865_4WIRE` → `MAX31865_3WIRE`. RREF 412.0Ω(실측값) 유지.
- **양 버전 적용.** (보드 솔더점퍼도 3선식인지 확인 필요)

### 4. 최대 RPM 80 제한 (탈조 한계)
- `MAX_OUTPUT_RPM` 100 → 80. 80 초과 시 탈조는 **마이크로스텝 오류가 아니라 토크-속도 한계**(출력 80=모터 ~297RPM에서 back-EMF로 토크 급감). 펄스 생성은 여유.
- 서버 clamp + 웹 UI(슬라이더/레시피입력/상태바) 동기. 구형 클라가 100 보내도 펌웨어가 80으로 clamp.
- **양 버전 적용.**

### 5. (디스플레이 버전 전용) 화면보호기 이미지 업로드
- 서버: 확장자 대신 **매직바이트**(GIF8/FFD8)로 타입 판정 + 실제 저장 성공 시에만 200 응답. iOS Safari가 filename 바꿔 업로드 실패하던 문제 해결.
- 클라이언트: 업로드 전 **320×240 강제 리사이즈 + 압축**(보드 부하 경감). GIF는 전 프레임 디코드→리사이즈→256색 양자화→프레임 간 차분 투명 최적화→GIF89a 재인코딩(애니메이션 유지). JPEG는 canvas 리사이즈+품질 압축.
- 무디스플레이 버전엔 화면보호기 없음 → 해당 없음.

## 하드웨어 검증 완료 (실측)

- **모터**: NEMA17 + 3.71:1 유성기어, 1.8°(200 step/rev), 정격 1.3A/상, 홀딩 0.45N·m.
- **드라이버**: TMC2209 스탠드얼론, StealthChop, **VREF 0.7V ≈ 0.5A**, 1/8 마이크로스텝.
- **전원**: 메인 20V / 보조배터리 12V.
- **냉각 (2존)**: 모터=수면 아래 PETG 벽 밀착 + 38°C 수조(세미 수랭) / 전자부=물 위 건식 공냉.
- **검증**: 75RPM 연속 2h+ → **모터·전자부 양쪽 ≈50°C.** PETG Tg 80°C 대비 30°C 마진. 발열 우선 정책상 VREF 최소(0.7V) 유지.

## 남은 오픈 항목

1. **오늘 수정분 실기 테스트** (양 버전 플래싱·동작 확인).

5. **HW 풀업 추가** — EN→3.3V 10K, TFT CS·RES→3.3V 10K (저항 도착 대기).
6. **모터 챔버 씰/결로 점검** — 물 침투 = 모터 즉사.
